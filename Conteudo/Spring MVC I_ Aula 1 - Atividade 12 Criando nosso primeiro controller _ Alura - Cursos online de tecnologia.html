Agora que já temos o Spring MVC no nosso projeto, não queremos mais fornecer arquivos estáticos, ou seja, queremos usar a estrutura MVC! Em outras palavras, criaremos o nosso primeiro controller. Então crie a classe HomeController dentro do pacote br.com.casadocodigo.loja.controllers. Em seguida, anote a HomeController com a annotation @Controller, para que o Spring MVC a reconheça como um controller.

Agora crie o método index() que será responsável em atender a home da nossa aplicação, ou seja, o endereço "/". Por fim, anote esse método com @RequestMapping("/") para atender a "/" e adicione um sysout com o valor System.out.println("Exibindo a home da CDC");.

Como formatar códigos

  • É possível criar blocos de código adicionando três backticks (```) antes e depois do código
  • Para escapar a formatação, utilize crases `dessa _forma_`
  • Serão exibidas apenas tags HTML e XML que estiverem entre ``

Formatação básica

  • **negrito**
  • *itálico*

Como formatar links

  • <http://google.com>
  • [Google](http://google.com)
  • <a href="http://google.com">Google</a>

O nosso código ficará parecido com a classe abaixo, mas repare que mesmo subindo o servidor ainda vamos ver a mensagem "Oi", pois não configuramos o Spring MVC para que ele fique responsável em atender as requisições recebidas pelo servidor, nesse caso, o Tomcat. Faremos isso no próximo exercício.

package br.com.casadocodigo.loja.controllers;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;

@Controller
public class HomeController {

    @RequestMapping("/")
    public void index() {
        System.out.println("Exibindo a home da CDC");
    }
}
